# Semantic ARIS - Тезисы

## A. Концепт

### Для пользователя
- **Внешне** — классическая ARIS-подобная система (аналог Toolset, Business Studio, Sparx Architect и т.п.)
- **Базовый функционал** — типовые компоненты:
  - Моделер (создание и редактирование схем)
  - Репозитарий (хранение объектов моделирования)
  - Паблишер (визуализация и публикация)

### Под капотом — Linked Data стек
- **Технологическая основа:** Semantic Web, Wikidata-подобные технологии
- **Хранилище:** Quadstore (triplestore) для RDF данных

### Внешние преимущества

#### 1. Абсолютный контроль над объектами моделирования
- Любой объект (процесс, роль, документ и т.п.), его атрибуты, группировки — всё доступно через SPARQL-запрос
- Не нужно программировать логику анализа — она декларативно описывается в запросах
- **Пример:** Построение матрицы RACI по SPARQL-запросу к схемам из репозитария
- **Для пользователей без знания SPARQL:** Конструктор запросов

#### 2. Механизм рассуждений (Semantic Reasoning)
- Запросы с учётом правил вывода через semantic reasoner
- **Пример:**
  - Утверждения: "Петя — сын Васи", "Вася — сын Вовы"
  - Reasoner автоматически выводит: "Петя — внук Вовы"
- Применение в BPM: автоматический вывод иерархии процессов, наследование атрибутов

#### 3. Арсенал LD-инструментов
- Использование существующих Linked Data инструментов для анализа и редактирования
- Совместимость с экосистемой Semantic Web

### Технологические преимущества (внутренние)

#### 1. Унификация и совместимость
- Все LD-системы моделирования обеспечивают высокий уровень совместимости
- Использование загружаемых онтологий для стандартизации

#### 2. Быстрота разработки
- Большой выбор библиотек (включая JavaScript):
  - N3.js — парсинг RDF
  - Comunica — SPARQL движок
  - In-memory quadstore
  - Semantic reasoner
- Готовые решения для triplestore, SPARQL, reasoning

#### 3. Прозрачность логики
- Semantic ARIS разработан на SPARQL (UI на JavaScript)
- Под каждой кнопкой — соответствующий SPARQL-запрос
- Абсолютно понятна логика работы каждой функции
- Простое добавление новых функций

## C. Ключевые технические принципы ver9d

### SPARQL-Driven Programming
- Бизнес-логика описывается декларативно через SPARQL-запросы
- JavaScript используется минимально — только для UI и координации
- Данные и их структура определяются онтологией
- Справочники и формы генерируются динамически на основе метаданных

### Архитектура хранилища
- **quadstore:** Хранение данных в формате квадов (subject, predicate, object, graph)
- **Графы:**
  - `vad:ptree` — дерево концептов процессов
  - `vad:rtree` — дерево концептов исполнителей
  - `vad:techtree` — технологическое приложение (методы, предикаты)
  - `vad:t_*` — схемы процессов (VADProcessDia)
  - `vad:vt_*` — виртуальные данные (автоматически вычисляемые)

### Модульная структура
- 12 функциональных модулей с чётким разделением ответственности
- Каждый модуль: UI (_ui.js), логика (_logic.js), SPARQL (_sparql.js)
