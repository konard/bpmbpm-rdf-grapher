В Archi модель состоит из concept - объектов со свойствами, именами и уникальными id.  
При построении диаграмм используются concept reference - визуальные представления, которые наследуют свойства и имена от concept, но дополнительно обладают свойствами оформления и своими уникальными id.  
По сути concept и concept reference - это равнозначные объекты, которые связаны друг с другом через свойства.  
Id присваивается случайным образом в момент создания объекта и представляет собой 128-битное число, что исключает вероятность повторов.  
Поле id доступно только для чтения, создание и существование двух объектов с одним id исключено.  

Concept - это термин в Archi, означающий объект в Model tree (непосредственно в модели), до переноса на View (диаграмму)

Где в archi почитать на эту тему? нигде. Эта информация добыта частично путём реверс-инжиниринга в процессе автоматизации работы, 
частично скомпилирована из Archi User Guide и документации по jArchi.

Тут же при создании обьекта где инструмент привязки его к концепту?  

Это происходит автоматически. При размещении на диаграмме процесса для него всегда создаётся concept reference. 
А, вот, будет ли при этом создан concept зависит от того как вы размещаете процесс. 
- Если вы переносите процесс из model tree, используется concept из model tree, нового не создаётся. 
- Если процесс взят из палитры, всегда создаётся и concept, и concept reference. 
- Если вы копируете процесс на диаграмме, у вас есть два варианта: вставить и concept, и concept reference, либо вставить только concept reference.   
Во всех случаях связь между concept и concept reference создаётся автоматически.  
Связь в последствии можно поменять.
