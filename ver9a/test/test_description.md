# Автоматизированное тестирование RDF Grapher ver9a

## Описание

Данный модуль предоставляет пример автоматизированного тестирования приложения RDF Grapher.
Тест загружает пример данных Trig_VADv5.ttl и выполняет автоматическую проверку корректности работы основных функций.

## Запуск

Откройте файл `test_auto.html` в браузере. Тесты выполняются автоматически при загрузке страницы.

## Состав тестов

1. **Тест парсинга RDF** - проверка корректного парсинга данных Trig_VADv5.ttl через N3.js
2. **Тест валидации VAD** - проверка соответствия данных правилам VAD онтологии
3. **Тест иерархии TriG** - проверка корректного построения иерархии TriG графов
4. **Тест вычисления подтипов** - проверка вычисления processSubtype для процессов
5. **Тест funSPARQLvalues** - проверка работы SPARQL-движка на простых запросах
6. **Тест getProcessIndividualsInTriG** - проверка получения индивидов процессов в TriG графе (issue #241)
7. **Тест getExecutorGroupsInTriG** - проверка получения групп исполнителей в TriG графе (issue #241)

## Ожидаемый результат

Все тесты должны завершиться со статусом "PASSED" (зелёный цвет).
При наличии ошибок тест показывает "FAILED" (красный цвет) с описанием проблемы.

## Расширение

Для добавления новых тестов:
1. Добавьте новую функцию `testXxx()` в файл `test_auto.html`
2. Добавьте вызов функции в массив `tests` в секции инициализации
3. Функция должна возвращать объект `{ passed: boolean, message: string }`
